Interfaces y Lenguajes para Diseño Web

Para poder entender lo que haremos tocaremos brevemente algunos conceptos. Estos pueden ser un poco confusos, pero es mejor estar tranquilos e ir poco a poco.

Diseño Web
Consiste en la planificación, diseño, implementación y mantenimiento de sitios web.

Web o www(world wide web)
Red mundial de recursos accesibles mediante la red de internet.

Sitio Web
Colección de páginas web relacionadas y comunes a un dominio de internet o subdominio en la web dentro de Internet.

S3

Inspección de una web

Al parecer poder cambiar cosas de una página web a primera vista es algo fácil. En este caso probamos con la página de El Comercio, donde pudimos identificar como esque se encuentra la información y como se define su estructura.

Plantillas web para una página

En internet nos podemos encontrar con innumerables plantillas que nos definan el diseño de nuestra página web, sin necesidad de tener total conocimiento de CSS y JavaScript, que son los que nos permiten diseñar y hacer que la página interactúe con su usuario.

En este momento estamos obsevando una, que ha sido modificada según mis necesidades.


Diagnóstico de fallas en Google Chrome para nuestras páginas web

Herramienta utilizada para detectar alguna falla dentro de la página web.

Diseño Web, estructura y arquitectura

Como ya lo hemos hablado, el sitio web alberga a todos los componentes de las páginas web, donde su proceso es lineal mientras que las páginas web tienen un desarrollo cíclico, que se basa en prueba-error.

El alojamiento web es el espacio donde la arquitectura web se desarrolla, y esta a la misma vez tiene su lenguaje, el HTML, que permite la comunicación entre sus componentes principales, el cliente y su usuario.

¿Y qué aprendimos esta semana?

El poder inspeccionar las páginas web y poder utilizar plantillas ya creadas para el diseño de las mismas, nos genera una ayuda para los que recién estamos iniciando en este mundo, nos da una guía y una matriz para evitar errores o solucionarlos, pero a la misma vez no quita la importancia de seguir aprendiendo y comprender en donde esque se materializa y como esque se configura.

La sesión inicio con un recuento de los temas tradados sobre Arquitectura Web e Infraestructura del HTML. Luego, empezamos a poner en marcha algún ejemplo de página web que hemos hecho en las sesiones anteriores, a través del uso de GitHub y Netlyfy. S4

Puesta en operación de una página web

Para esto tenemos que tener en cuenta el contenido que vamos a presentar (estructura, contenido y recursos), donde va a estar alojado (carácterisicas del espacio físico y características funcionales que cumplan con nuestras necesidades) y el nombre del dominio que usaremos para esta.

Alojamiento Web

Conocido como "hosting" o "web hosting". Tenemos que tener en cuenta antes de seleccionar alguno: costo, panel de control, tráfico mensual y cantidad de conexiones y servicios complementarios.

Aplicaciones

Para poder publicar un página web sin costo es necesario de un Serverless (sin servidor). Para el ejemplo de esta sesión utilizamos Netlify, pero existen otras como: GitHube Pages.

¿Y qué aprendimos esta semana?

En esta sesión pudimos conocer más a fondo de como ya ingresar nuestra página web dentro de un servidor con ayuda de aplicaciones gratuitas con las cuales ya pudimos materializarla y poder tener un html. Además de, tener en cuenta cosas más específicas, como nuestro presupuesto, para cuanta gente esta pensado, etc., para escoger el hosting propicio para nuestro proyecto. Además, hablando de nuestro proyecto innovador expuesto, es indispensable que la solución este completamente ligada al desarrollo de una problemática.
